# Fig Editor Goal Coverage

This matrix is the working checklist for the requested fig renderer/editor parity work.
`fig` domain data is the SoT; SVG, React shell, and WebGL must consume the same
SceneGraph/RenderTree or explicit domain adapters.

| Area | Goal | Status | Verification |
| --- | --- | --- | --- |
| WebGL test harness | WebGL specs live under `packages/@aurochs-renderer/fig/spec/renderer/webgl` with shared harness | Done | `bun run --cwd packages/@aurochs-renderer/fig test:webgl` |
| Effect SoT | Fig effects are classified once and consumed by SVG/React/WebGL | Done | effect coverage specs + WebGL effect attribute specs |
| WebGL effect coverage | Drop shadow, inner shadow, layer blur, background blur requirements are measurable | Done | effect requirement registry specs |
| Renderer selection | React editor shell selects SVG or WebGL backend, not a third renderer path | Done | `FigPageRenderer.spec.tsx` |
| User intent/operation SoT | Canvas behavior derives from one resolved user intent and one permitted-operation domain object, not scattered `creationMode` / `textEdit` / `drag` checks | Done | `user-intent.spec.ts` fixes precedence: text edit > active drag > toolbar tool. `user-operation.spec.ts` verifies which operations are allowed for text edit, path edit, select, creation, and transform intents. `interaction-policy.spec.ts` verifies derived UI affordances. `FigEditorCanvas` consumes the operation domain before dispatching selection, creation, transform, text-edit, context-menu, and path-edit actions. Keyboard shortcuts are registered from the canvas so drag-aware operation gating does not re-render the shell. Toolbar/inspector/panel entry points also check the operation domain before tool/selection/outline operations. |
| Mutation source guard | Inspector, layer panel, text editor, path editor, and canvas menu mutations are tagged and rejected at reducer entry when the current user intent does not permit them | Done | `UPDATE_NODE` / `ADD_IMAGE_ASSET` / `RENAME_NODE` require an explicit mutation source. `action-guard.ts` has no fallback allow branch and maps document/page/node/selection/drag/create/clipboard/history actions to `FigUserOperation`. `action-guard.spec.ts` verifies property-panel mutations are rejected during text edit, path edit, and active canvas transform while the intended text/path/drag operations still pass. |
| Property panel operation UX | Inspector controls consume the same operation domain and remain editable only when property mutation is permitted | Done | `PropertyPanel` resolves operation domain from `creationMode` / `textEdit` / real `drag` state and disables property mutation scopes when `update-property` is not allowed. Every property section consumes `PropertyMutationTarget`; multi-target sections dispatch `UPDATE_NODES`, while primary-only sections dispatch through `createPropertyPrimaryUpdateAction` so raw `UPDATE_NODE` cannot reappear in section code. `property-mutation-target.spec.ts` fails if a section dispatches raw node updates. `property-panel-operation-domain.e2e.ts` uses aria roles and mouse/keyboard actions to verify transform X editing actually moves the selected rectangle, multi-selection fill edits recolor every selected rendered node, and text edit, vector edit, and active canvas transform disable inspector mutation controls. |
| Layer/page/toolbar operation UX | Layer, page, toolbar, and inspector-tree entry points consume the same current operation domain instead of idle assumptions | Done | `useFigOperationDomain()` resolves the domain from editor state plus real drag state. `PageListPanel` disables page edits, `LayerPanel` gates layer selection/rename, `FigEditorToolbar` disables tools/undo/redo from real drag state, and `FigInspectorPanel` selection uses the same hook. `layer-page-operation-domain.e2e.ts` verifies layer selection and rename work in select mode, and page mutation / layer rename are disabled during text edit, vector edit, and active canvas transform. |
| Path overlay constants SoT | Vector path hit sizes, handle sizes, colors, and dash scaling are not hard-coded in canvas handlers | Done | `vector-path-overlay-style.ts` owns path overlay visual/hit constants; `vector-path-overlay-style.spec.ts` verifies screen-pixel scaling across zoom. |
| ViewBox model | SVG/WebGL consume one SceneGraph viewport window over a larger canvas | Done | `FigPageRenderer.spec.tsx` + browser smoke |
| Rulers on fig canvas | Rulers show real page coordinates, including negative/off-window coordinates | Done | `SvgRulers.spec.tsx`; fig canvas uses unbounded mode |
| WebGL DPR/zoom | Zoom changes do not recreate renderer/scene and respect bounded DPR | Done | `FigWebGLViewportCanvas` keeps renderer/scene refs; `viewport-pixel-ratio.spec.ts` covers bounded quantized backing-store ratio |
| Texture resource SoT | Texture identity is explicit domain resource, not ad hoc keys | Done | WebGL renderer/cache consume `TextureResource`; `texture-resource.spec.ts` |
| Text fallback ban | WebGL does not hide missing text behind Canvas2D fallback | Done | no Canvas2D text texture path; text resolver specs |
| WebGL text outlines | WebGL receives glyph contours from derived data or explicit preloaded font cache | Done | `text-font-resolver.spec.ts` verifies SceneGraph/RenderTree glyph mode from explicit font resolver |
| SVG/WebGL text wrapping | SVG and WebGL line breaks derive from shared text layout | Done | fallback wrap removed; `text-font-resolver.spec.ts` verifies wrapped `textLineLayout` is preserved into RenderTree source/content |
| Text edit overlay | Editor text input does not white-fill the underlying render | Done | browser WebGL check: transparent textarea + no white SVG rect |
| Internal-only canvases | User page lists hide canvases via Kiwi metadata, never by name | Done | domain conversion spec + renderer-debug raw canvas spec |
| Frame/Component/Symbol editing | Creation, grouping, componentization, symbolization work inside containers | Done | reducer specs + Chromium smoke |
| Shape creation | Rect, oval, shape creation works in canvas and nested containers | Done | reducer specs + Chromium frame smoke |
| Text insertion/editing | Insert/edit text positions match backend render, including WebGL | Done | `computeTextLayout` exposes source-ranged visual lines and SVG/WebGL/editor overlays consume that layout directly. `text-editing.e2e.ts` now runs in the main interaction suite and verifies double-click entry, typing, Backspace/Delete separation, Cmd+A, Escape, focus ownership, shortcut isolation, caret/frame/selection chrome, and drag selection through mouse/keyboard actions. `svg-text-edit-visual-parity.e2e.ts` and `webgl-text-edit.e2e.ts` verify wrapped source selection maps to the rendered visual line across SVG/WebGL. |
| Bounding box | Selection box does not scale stroke width with zoom | Done | `SelectionBox.spec.tsx` checks inverse scale stroke/handles and `vector-effect` |
| Rotation UX | Rotate from selection edge/corner, not only top arrow | Done | `SelectionBox.spec.tsx` edge hit zones; `drag-handlers.spec.ts` checks multi-selection bounding-box rotation |
| Origin/layout controls | Origin, constraints, fit direction, autolayout sizing are editable | Done | AutoLayout/constraints domain fields covered by section specs; `transformOrigin` is modeled in fig domain, roundtripped, editable in Transform section, and consumed by panel + drag rotation SoT |
| Exports | Figma-like export settings/actions are available where implementable | Done | `.fig` download helper + node export presets in `ExportSettingsSection.spec.tsx`; domain roundtrip preserves `exportSettings` |
| Layer list | Rename, type decoration, background styling for frame/component/symbol | Done | rename UI in `LayerPanel`; presentation SoT covered by `layer-node-presentation.spec.ts` |
| Path editing | Bezier point editing, point addition, path drawing, basic-shape topology edits, outline conversion, boolean operations from canvas context menu | Done | Vector Edit is a first-class toolbar/keyboard tool. VECTOR nodes and basic shapes (rectangle, rounded rectangle, ellipse, line, polygon, star) enter the same explicit path-edit flow. Selected paths expose canvas anchor/control handles, control lines, segment hit regions, nearest-segment point insertion, handle dragging, segment line<->curve conversion, anchor deletion, and open/close path commands. New vector paths can be drawn by dragging in Vector Edit, including inside frames; the parent frame is used only as insertion scope and does not absorb the pointer operation. Basic shapes keep parametric resize behavior for simple anchor drags, and segment/topology edits explicitly commit them to editable vector paths instead of silently doing only bounding-box changes. Path editing renders in the EditorCanvas interaction overlay above item hit areas and selection chrome, so frame/frame-in-frame hit areas cannot absorb vector handles or path segments. Union/Subtract/Intersect/Exclude are wired as `BOOLEAN_OPERATION` editor actions and context-menu commands. Browser coverage verifies toolbar path editing, topmost vector handle/segment targets, frame-in-frame path editing, basic-shape path editing, basic-shape topology point insertion, new path drawing inside a frame, point insertion, curve conversion, line conversion, point deletion, zoom-stable selection chrome, frame child selection, and boolean menu flow. |
| Overrides | Override editing form covers editable override kinds | Done | component property BOOL/TEXT/VARIANT/NUMBER/INSTANCE_SWAP/COLOR/IMAGE/SLOT editors covered; domain component properties write back to Kiwi `componentProp*` fields |
| Fill/stroke assets | Solid, gradient, and image fills/strokes editable from UI | Done | solid/gradient/image asset registry/scale/rotation controls covered by fill/stroke and image asset specs |
| Multi-turn persistence | Edits remain fixed across repeated operations | Done | `multi-turn-persistence.spec.ts` covers repeated move and text updates |

## Kiwi-backed operation checklist

This table tracks first-class fig/Kiwi fields that must not be treated as
inspector-only metadata. A row is done only when the value can be changed through
human UI controls and the rendered/editor state is verified after another
selection turn.

| Kiwi/domain area | Fields / values | UI operation | Persistence/render verification | Status |
| --- | --- | --- | --- | --- |
| Component instance properties | `componentPropertyDefs`, `componentPropertyRefs`, `componentPropertyAssignments` for `BOOL`, `TEXT`, `INSTANCE_SWAP` | Property panel controls with accessible labels | `component-section-kiwi-operations.e2e.ts` edits label text, swaps the icon component, toggles icon visibility, checks SVG render, then reselects the instance and verifies controls retain values | Done |
| Component reference properties | `VARIANT`, `SLOT` reference values | Reference select consumes the component registry instead of writing text payloads | `ComponentPropertiesSection.spec.tsx` verifies `VARIANT` is emitted as a reference select, not `textValue` | Done |
| Component reference metadata | `COLOR`, `IMAGE` reference values | Explicit reference input, no value-shape fallback | `ComponentPropertiesSection.spec.tsx` verifies accessible inputs preserve reference values | Done |
| Section behavior | `sectionContentsHidden` | Section property switch | `component-section-kiwi-operations.e2e.ts` toggles the switch and verifies the value survives selection turns; builder context specs verify raw Kiwi roundtrip | Done |
| Auto-layout child controls | `stackChildAlignSelf`, `stackChildPrimaryGrow` | Layout constraints section | `document-to-tree.spec.ts` and `tree-to-document.spec.ts` verify domain <-> Kiwi roundtrip so UI edits are not dropped on export | Done |
| Component sets / variants | `COMPONENT_SET`, `variantPropSpecs`, variable-driven variant resolution | Variant authoring and variant-set editing UI | Not yet implemented in the editor operation surface | Open |
| Override authoring | `symbolOverrides`, nested override path targeting, style/geometry override entries | Override-path editor beyond component property assignments | Renderer/domain resolution exists, but direct authoring UI is not complete | Open |
